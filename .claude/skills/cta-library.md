# CTA Library - CTA 문구 라이브러리

## 개요
채널별, 목적별 CTA(Call-to-Action) 문구 모음입니다.
자연스럽고 강요하지 않는 톤을 유지합니다.

---

## 링크드인 CTA

### 댓글 유도
```
💬 가벼운 질문
- "여러분은 어떻게 하고 계세요?"
- "혹시 비슷한 경험 있으신가요?"
- "다른 방법 있으면 알려주세요!"

💬 선택형 질문
- "A vs B, 어떤 게 더 효과적일까요?"
- "1번과 2번 중 뭐가 더 공감되세요?"

💬 경험 공유 요청
- "써보신 분 후기 궁금해요"
- "실제로 해보신 분 어떠셨어요?"
```

### 저장/공유 유도
```
📌 저장 유도
- "나중에 필요할 때 다시 보세요"
- "저장해두면 언젠간 쓸 일 있을 거예요"
- "필요한 분께 공유해주세요"

🔄 공유 유도
- "도움이 됐다면 동료에게도 공유해주세요"
- "팀원들과 나눠보세요"
```

### 팔로우 유도
```
👋 자연스러운 팔로우
- "이런 팁 더 보고 싶으시면 팔로우해주세요"
- "매주 [주제] 관련 글 올려요"
- "비슷한 고민하시는 분들 환영해요"
```

## X CTA

### 리트윗 유도
```
🔁 공유 요청
- "도움됐으면 RT"
- "필요한 사람 태그해주세요"
- "저장 필수"

🔁 가치 강조
- "이거 알면 시간 아낌"
- "나중에 또 찾을 정보"
```

### 팔로우 유도
```
👋 간결하게
- "팔로우하면 더 많은 팁"
- "이런 거 매일 올려요"
- "프로필에 더 많은 정보"
```

### 스레드 연결
```
🧵 스레드 유도
- "스레드로 자세히 풀어볼게요 👇"
- "아래에서 계속"
- "1/"
```

### 대화 유도
```
💬 짧은 질문
- "어떻게 생각해요?"
- "써봤어요?"
- "궁금한 거 물어보세요"
```

## 뉴스레터 CTA

### 회신 유도
```
📧 대화 시작
- "이 메일에 바로 답장 주셔도 돼요"
- "궁금한 점 있으면 회신해주세요"
- "여러분 생각이 궁금해요. 편하게 답장 주세요"

📧 피드백 요청
- "오늘 내용 어떠셨어요? 솔직한 피드백 환영해요"
- "이런 주제 더 다뤘으면 좋겠다 싶은 거 있으면 알려주세요"
```

### 공유 유도
```
🔗 포워딩 요청
- "도움이 됐다면 주변에 공유해주세요"
- "이 뉴스레터 유용하다고 느끼셨다면, 필요한 분께 전달해주세요"

🔗 구독 유도 (공유용)
- "아직 구독 전이라면: [구독 링크]"
```

### 다음 예고
```
📅 기대감 조성
- "다음 주에는 [주제]에 대해 다룰게요"
- "더 자세한 내용은 다음 레터에서"
- "곧 [새로운 것] 공개할 예정이에요. 기대해주세요!"
```

### 액션 아이템
```
✅ 실행 유도
- "오늘 딱 한 가지만 해보세요: [구체적 행동]"
- "이번 주 미션: [행동]"
- "지금 바로 [행동] 해보세요"
```

## 목적별 CTA 선택 가이드

| 목적 | 링크드인 | X | 뉴스레터 |
|------|---------|---|---------|
| 인게이지먼트 | 댓글 질문 | RT 요청 | 회신 유도 |
| 성장 | 팔로우 유도 | 팔로우 + 알림 | 공유/구독 |
| 가치 전달 | 저장 유도 | 저장 + 스레드 | 액션 아이템 |
| 관계 구축 | 경험 공유 요청 | 대화 유도 | 피드백 요청 |

## CTA 작성 원칙

### DO ✅
```
- 구체적으로: "댓글로 알려주세요" > "참여해주세요"
- 가치 먼저: "도움이 됐다면" + CTA
- 쉽게: 한 가지 행동만 요청
- 자연스럽게: 본문과 연결되게
```

### DON'T ❌
```
- 강요하지 않기: "반드시 ~해주세요"
- 여러 개 동시에: "좋아요, 댓글, 공유, 저장해주세요"
- 뻔한 표현: "구독과 좋아요 부탁드려요"
- 과한 요청: 첫 글에서 "팔로우해주세요"
```

## 상황별 CTA 예시

### 정보/팁 공유 후
```
링크드인: "여러분만의 팁도 있으면 댓글로 공유해주세요"
X: "저장해두면 나중에 씀"
뉴스레터: "오늘 하나만 실천해보세요"
```

### 경험담 공유 후
```
링크드인: "비슷한 경험 있으신가요?"
X: "공감되면 RT"
뉴스레터: "여러분의 경험도 들려주세요"
```

### 도구/리소스 소개 후
```
링크드인: "써보신 분 후기 궁금해요"
X: "써본 사람?"
뉴스레터: "사용해보고 어땠는지 회신 주세요"
```

### 인사이트 제시 후
```
링크드인: "어떻게 생각하세요?"
X: "동의? 반대?"
뉴스레터: "다른 의견 있으면 편하게 답장 주세요"
```
