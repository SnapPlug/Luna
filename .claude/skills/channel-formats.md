# Channel Formats - 채널별 규격

## 개요
각 채널(링크드인, X, 뉴스레터)의 형식과 규격을 정의합니다.

---

## 링크드인 (LinkedIn)

### 기본 규격
| 항목 | 규격 |
|------|------|
| 글자 수 | 800~1,300자 |
| 최대 | 3,000자 (비권장) |
| 줄바꿈 | 2~3문장마다 |
| 해시태그 | 3~5개 |

### 구조 템플릿
```
[훅 - 1~2줄, 강렬하게]

[문제/공감 - 2~3줄]

[해결책/인사이트 - 본문]
- 포인트 1
- 포인트 2
- 포인트 3

[실제 사례/수치]

[마무리 + CTA]

---
#해시태그1 #해시태그2 #해시태그3
```

### 특징
- "입니다/습니다" 체 혼용 가능
- 전문성과 친근함 밸런스
- 스크롤 멈추게 하는 첫 줄 중요
- "더 보기" 클릭 유도 (처음 150자 승부)

### 해시태그 예시
```
#AI활용 #생산성 #자동화 #1인창업 #스타트업
#업무효율 #ChatGPT #노코드 #솔로프리너
```

## X (Twitter)

### 기본 규격
| 항목 | 규격 |
|------|------|
| 글자 수 | 280자 이내 |
| 권장 | 200~250자 |
| 줄바꿈 | 자유롭게 |
| 해시태그 | 1~2개 (선택) |

### 구조 템플릿

#### 단일 트윗
```
[훅 - 한 줄]

[핵심 메시지 - 2~3줄]

[CTA 또는 여운]
```

#### 스레드 (필요시)
```
1/ [메인 훅 + 예고]

2/ [포인트 1]

3/ [포인트 2]

4/ [포인트 3]

5/ [마무리 + CTA]
```

### 특징
- 임팩트 있는 한 줄이 핵심
- 숫자/이모지로 시선 끌기
- 리트윗 유도하는 인사이트
- 대화체, 반말도 가능

### 효과적인 훅 패턴
```
- "~하는 사람 손?"
- "솔직히 말할게요."
- "[숫자]개월 동안 [행동]했더니"
- "이거 모르면 손해예요"
```

## 뉴스레터 (Newsletter)

### 기본 규격
| 항목 | 규격 |
|------|------|
| 글자 수 | 1,500~3,000자 |
| 읽기 시간 | 3~7분 |
| 섹션 | 3~5개 |
| 소제목 | 필수 |

### 구조 템플릿
```markdown
# [메인 제목]

> [한 줄 요약 또는 인용구]

안녕하세요, [인사말]

[도입 - 이번 주제 소개, 왜 중요한지]

---

## 1. [첫 번째 포인트]
[상세 설명]
[예시 또는 사례]

## 2. [두 번째 포인트]
[상세 설명]
[예시 또는 사례]

## 3. [세 번째 포인트]
[상세 설명]
[예시 또는 사례]

---

## 💡 바로 적용해보세요
[실행 가능한 액션 아이템]

---

## 마무리
[요약 + 다음 예고]

[CTA - 회신 유도, 공유 요청 등]

[서명]
```

### 특징
- 스토리텔링 가능
- 깊이 있는 설명
- 개인적인 톤 유지
- 독자와의 관계 구축 중요

### 섹션별 팁
- **제목**: 열어보고 싶게 (호기심 자극)
- **도입**: 3문장 안에 가치 전달
- **본문**: 스킵해도 이해되게 (소제목만 봐도)
- **마무리**: 행동 유도 명확하게

---

## 채널별 비교표

| 항목 | 링크드인 | X | 뉴스레터 |
|------|---------|---|---------|
| 길이 | 800~1,300자 | ~280자 | 1,500~3,000자 |
| 톤 | 전문적+친근 | 캐주얼 | 개인적+깊이 |
| 말투 | 입니다/요 혼용 | 요/반말 | 요체 |
| 구조 | 리스트형 | 임팩트형 | 스토리형 |
| 해시태그 | 3~5개 | 0~2개 | 없음 |
| CTA | 댓글/좋아요 | RT/팔로우 | 회신/공유 |

---

## 글자 수 계산 규칙

### 포함
- 본문 텍스트
- 이모지 (1개 = 1자)
- 줄바꿈 (1개 = 1자)

### 미포함
- 해시태그 (별도 카운트)
- 구분선 (---)
- 링크 URL
