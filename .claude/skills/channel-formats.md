# Channel Formats - 채널별 규격

## 개요
각 채널(링크드인, X, 뉴스레터)의 형식과 규격을 정의합니다.

---

## 링크드인 (LinkedIn)

### 기본 규격
| 항목 | 규격 |
|------|------|
| 글자 수 | 800~1,300자 |
| 최대 | 3,000자 (비권장) |
| 줄바꿈 | 2~3문장마다 |
| 해시태그 | 3~5개 |

### 구조 템플릿
```
[훅 - 1~2줄, 강렬하게]

[문제/공감 - 2~3줄]

[해결책/인사이트 - 본문]
- 포인트 1
- 포인트 2
- 포인트 3

[실제 사례/수치]

[마무리 + CTA]

---
#해시태그1 #해시태그2 #해시태그3
```

### 특징
- "입니다/습니다" 체 혼용 가능
- 전문성과 친근함 밸런스
- 스크롤 멈추게 하는 첫 줄 중요
- "더 보기" 클릭 유도 (처음 150자 승부)

### 해시태그 예시
```
#AI활용 #생산성 #자동화 #1인창업 #스타트업
#업무효율 #ChatGPT #노코드 #솔로프리너
```

## X (Twitter)

### 기본 규격
| 항목 | 규격 |
|------|------|
| 글자 수 | 280자 이내 |
| 권장 | 200~250자 |
| 줄바꿈 | 자유롭게 |
| 해시태그 | 0~2개 (선택) |

### X 알고리즘 핵심
X의 For You 피드는 Grok 기반 ML 모델이 참여도를 예측하여 노출을 결정합니다.

**참여 행동 가중치 (높은 순)**:
1. 💬 답글 - 최고 가중치
2. 🔄 리포스트 - 매우 높음
3. 💭 인용 - 높음
4. ❤️ 좋아요 - 중간
5. 🖱️ 클릭/조회 - 기본

### 구조 템플릿

#### 단일 트윗
```
[훅 - 한 줄, 스크롤 멈추게]

[핵심 메시지 - 2~3줄]

[참여 유도 CTA]
```

#### 답글 유도형
```
[충격/질문 훅]

[인사이트 or 경험]

여러분 생각은요? 👇
```

#### 리포스트 유도형
```
[가치 있는 정보 예고]

[핵심 팁 3개]

저장해두면 쓸모있어요 🔖
```

#### 스레드 (필요시)
```
1/ [메인 훅 + 예고]
스레드로 정리했어요 👇

2/ [포인트 1]

3/ [포인트 2]

4/ [포인트 3]

5/ [마무리 + 참여 유도]
```

### 특징
- 첫 줄이 전부 (타임라인 노출 영역)
- 진정한 참여 유도가 핵심
- 답글/리포스트/인용 유도 설계
- 대화체, 반말도 가능
- 위트 + 진정성

### 알고리즘 최적화 훅 패턴
```
답글 유도: "~하는 사람 손?", "여러분 생각은요?"
리포스트 유도: "저장해두면 쓸모있어요", 체크리스트형
인용 유도: "논쟁적일 수 있는데...", 도발적 의견
```

### 참여 유도 CTA
```
답글: "비슷한 경험 있으신 분?", "다른 방법 알면 알려주세요!"
리포스트: "도움이 됐다면 RT 🙏", "주변에 필요한 분 있으면 공유해주세요"
인용: "여러분 케이스도 인용으로 알려주세요", "반박 환영합니다 😄"
```

### 금지 사항 (알고리즘 부정 신호)
```
❌ 클릭베이트 (기대 vs 실제 내용 괴리)
❌ 과도한 해시태그/이모지 (3개 이상)
❌ 스팸성 반복 게시
❌ "팔로우하면 ~" 강요
❌ 링크만 던지기
```

## 뉴스레터 (Newsletter)

### 기본 규격
| 항목 | 규격 |
|------|------|
| 글자 수 | 1,500~3,000자 |
| 읽기 시간 | 3~7분 |
| 섹션 | 3~5개 |
| 소제목 | 필수 |

### 구조 템플릿
```markdown
# [메인 제목]

> [한 줄 요약 또는 인용구]

안녕하세요, [인사말]

[도입 - 이번 주제 소개, 왜 중요한지]

---

## 1. [첫 번째 포인트]
[상세 설명]
[예시 또는 사례]

## 2. [두 번째 포인트]
[상세 설명]
[예시 또는 사례]

## 3. [세 번째 포인트]
[상세 설명]
[예시 또는 사례]

---

## 💡 바로 적용해보세요
[실행 가능한 액션 아이템]

---

## 마무리
[요약 + 다음 예고]

[CTA - 회신 유도, 공유 요청 등]

[서명]
```

### 특징
- 스토리텔링 가능
- 깊이 있는 설명
- 개인적인 톤 유지
- 독자와의 관계 구축 중요

### 섹션별 팁
- **제목**: 열어보고 싶게 (호기심 자극)
- **도입**: 3문장 안에 가치 전달
- **본문**: 스킵해도 이해되게 (소제목만 봐도)
- **마무리**: 행동 유도 명확하게

## 채널별 비교표

| 항목 | 링크드인 | X | 뉴스레터 |
|------|---------|---|---------|
| 길이 | 800~1,300자 | ~280자 | 1,500~3,000자 |
| 톤 | 전문적+친근 | 캐주얼 | 개인적+깊이 |
| 말투 | 입니다/요 혼용 | 요/반말 | 요체 |
| 구조 | 리스트형 | 임팩트형 | 스토리형 |
| 해시태그 | 3~5개 | 0~2개 | 없음 |
| CTA | 댓글/좋아요 | RT/팔로우 | 회신/공유 |

## 글자 수 계산 규칙

### 포함
- 본문 텍스트
- 이모지 (1개 = 1자)
- 줄바꿈 (1개 = 1자)

### 미포함
- 해시태그 (별도 카운트)
- 구분선 (---)
- 링크 URL
